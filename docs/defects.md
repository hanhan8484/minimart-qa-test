# MiniMart 缺陷報告（Defect Log）

- 產品：MiniMart v2.0
- 依據：`PRD.md`（與 PRD 不符即為缺陷）
- 用途：自動化／手動測試發現的缺陷先登記於此，再視需要開 GitHub Issue 或回報開發
- 更新方式：新增一列到「缺陷清單」；狀態變更時改「狀態」欄並在「歷程」補一行

---

## 狀態定義

| 狀態 | 說明 |
|---|---|
| **Open** | 已確認，尚未修復 |
| **Confirmed** | 已複驗，確定可重現 |
| **Fixed** | 開發已修，待回歸 |
| **Verified** | 回歸通過，可關閉 |
| **Won't Fix** | 不修復（需註明原因） |
| **Duplicate** | 與其他缺陷重複 |

## 嚴重度

| 等級 | 說明 |
|---|---|
| **S1 Critical** | 無法登入／下單／資料損毀 |
| **S2 Major** | 主流程受阻或金額／庫存錯誤 |
| **S3 Minor** | 文案、UI 小錯、非主路徑 |
| **S4 Trivial** | 錯字、樣式、體驗小瑕疵 |
