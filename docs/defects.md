- 更新方式：新增一列到「缺陷清單」；狀態變更時改「狀態」欄並在「歷程」補一行

---

## 狀態定義

| 狀態 | 說明 |
|---|---|
| **Open** | 已確認，尚未修復 |
| **Confirmed** | 已複驗，確定可重現 |
| **Fixed** | 開發已修，待回歸 |
| **Verified** | 回歸通過，可關閉 |
| **Won't Fix** | 不修復（需註明原因） |
| **Duplicate** | 與其他缺陷重複 |

## 嚴重度

| 等級 | 說明 |
|---|---|
| **S1 Critical** | 無法登入／下單／資料損毀 |
| **S2 Major** | 主流程受阻或金額／庫存錯誤 |
| **S3 Minor** | 文案、UI 小錯、非主路徑 |
| **S4 Trivial** | 錯字、樣式、體驗小瑕疵 |

| **嚴重度** | S2 Major |
| **相關 PRD** | R-1.7（點擊「登出」後，系統清空購物車內所有商品，並回到登入頁。再次登入時購物車為空。） |
| **環境** | 受測站（`BASE_URL`；見 README／`.env.example`） |

| **實際結果** | 購物車品項仍在（例如仍有「手沖咖啡濾杯 × 1」） |

| **附件／備註** | UI 登出會把導覽徽章清成 0，但伺服器 cart 未清，易造成徽章與車頁短暫不一致 |
