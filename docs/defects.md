### DEF-001｜導覽列「我的優惠卷」錯字（應為「優惠券」）

| 欄位 | 內容 |
|---|---|
| **狀態** | Open |
| **嚴重度** | S4 Trivial |
| **相關 PRD** | R-1.3（導覽列由左至右含「我的優惠券」） |
| **環境** | `https://cand1.tail296b14.ts.net/`（MiniMart v2.0） |
| **發現方式** | Batch1 自動化 `G-B02`；`page.locator('header.navbar a')` 實際文案為「我的優惠卷」 |
| **重現步驟** | 1. 以 `demo@minimart.test` 登入<br>2. 檢視頂部導覽列「優惠券」連結文案 |
| **期望結果** | 顯示「我的優惠券」（券） |
| **實際結果** | 顯示「我的優惠卷」（卷） |

| **自動化處理** | `tests/helpers/navbar.ts` 暫時接受 `/我的優惠[券卷]/`，避免誤報；修復後應改回只接受「券」 |
| **附件／備註** | — |

### DEF-002｜登出後伺服器購物車未清空

| 欄位 | 內容 |
|---|---|
| **狀態** | Confirmed |
| **嚴重度** | S2 Major |
| **相關 PRD** | R-1.7（點擊「登出」後，系統清空購物車內所有商品，並回到登入頁。再次登入時購物車為空。） |
| **環境** | `https://cand1.tail296b14.ts.net/`（MiniMart v2.0） |
| **發現方式** | Batch2 自動化 `G-C01`；另以 `/api/cart` 在登出再登入後複驗 |
| **重現步驟** | 1. 登入並加入至少 1 件商品<br>2. 確認 `GET /api/cart` 有品項<br>3. 點「登出」<br>4. 再次登入<br>5. 再呼叫 `GET /api/cart` |
| **期望結果** | 再次登入後購物車為空（`count=0` / `items=[]`） |
| **實際結果** | 購物車品項仍在（例如仍有「手沖咖啡濾杯 × 1」） |
| **影響** | 使用者以為已登出清空，實際下次登入仍看到舊購物車；違反登出語意 |
| **自動化處理** | `tests/flows/g-logout-clears-cart.spec.ts` 以 `test.fail(true, 'DEF-002…')` 標記預期失敗；修復後移除 `test.fail` |
| **附件／備註** | UI 登出會把導覽徽章清成 0，但伺服器 cart 未清，易造成徽章與車頁短暫不一致 |

**歷程**

| 日期 | 動作 |
|---|---|
| 2026-07-17 | 開單；API 複驗 Confirmed |

### DEF-003｜未滿 10 分鐘的待出貨訂單無「取消訂單」

| 欄位 | 內容 |
|---|---|
| **狀態** | Confirmed |
| **嚴重度** | S2 Major |
| **相關 PRD** | R-6.5、R-14.9（待出貨且下單未滿 10 分鐘應顯示「取消訂單」；確認後→「已取消」） |
| **環境** | `https://cand1.tail296b14.ts.net/`（MiniMart v2.0） |
| **發現方式** | Batch3 探測／自動化 `O-C02`；新下單後開詳情 |
| **重現步驟** | 1. Reset 後登入<br>2. 加入商品並完成結帳<br>3. 點「查看訂單」進入剛成立的待出貨詳情<br>4. 檢視「狀態與操作」區塊與 `GET /api/orders/{id}` |
| **期望結果** | 顯示「取消訂單」；點擊後對話框「確定要取消這筆訂單嗎？」；確認後狀態「已取消」 |
| **實際結果** | 僅有「模擬出貨（Demo）」；無「取消訂單」。訂單 JSON 有 `canShip`／`canConfirmReceipt`／`canApplyReturn`，**無** `canCancel`。`POST /api/orders/{id}/cancel` → 404 |
| **影響** | 買家無法依 PRD 取消新單；庫存／優惠券回補路徑無法從 UI 驗證 |
| **自動化處理** | `tests/flows/o-cancel-order.spec.ts` 以 `test.fail(true, 'DEF-003…')` 標記預期失敗；修復後移除 |
| **附件／備註** | A.4 第三筆因超過 10 分鐘無取消鈕屬正確行為；本缺陷針對**剛下單**的待出貨單 |

**歷程**

| 日期 | 動作 |
|---|---|
| 2026-07-17 | 開單；UI＋API 複驗 Confirmed |
